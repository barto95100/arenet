Conformité sanitizer

  • ✓
    Aucun élément interdit — pas de <main>, <section>, <article>, <header>, <footer>, <nav>, <svg>, <button>, <script>, <iframe>, <form>, <input>
  • ✓
    Aucun event handler inline — pas de onclick, onload, onerror ni href="javascript:"
  • ✓
    Structure — <div class="content"> avec z-index:5 au lieu de <main>
  • ✓
    Actions — <a href="/console"> stylé comme bouton, jamais <button>
  • ✓
    Animations — CSS @keyframes uniquement, aucun JS, aucun SVG. Les 12 particules sont des <div> hardcodés avec :nth-child(n) pour leurs vecteurs
  • ✓
    CSS — bloc <style> dans <head> préservé via AllowUnsafe(true), supporte @keyframes, CSS vars, @media, oklch(), pseudo-elements
  • ✓
    Fonts — system stacks (system-ui, sans-serif) — pas de Google Fonts CDN pour éviter l'injection nofollow par le sanitizer
  • ✓
    Fallback couleur — pas de background-clip:text + -webkit-text-fill-color:transparent sur le code principal (texte resterait invisible si CSS var ne charge pas)

Utilisation dans Arenet

Arenet sert ces pages via son Caddy embarqué, configuré en JSON généré automatiquement — il n'y a pas de Caddyfile à éditer. Ces fichiers HTML sont des gabarits à coller directement dans l'interface d'administration.

Pour les utiliser : Réglages → Pages d'erreur, créez un template, puis collez le corps HTML de chaque fichier dans l'onglet du code de statut correspondant (401 / 403 / 404 / 429 / 500 / 502 / 503 / 504).

Les placeholders ({http.error.id}, {http.request.uuid}, etc.) sont substitués par Caddy directement au moment de servir la réponse — aucune configuration supplémentaire n'est nécessaire.

Attachez ensuite ce template à une route (ou définissez-le comme gabarit par défaut catch-all) depuis l'éditeur de route.
