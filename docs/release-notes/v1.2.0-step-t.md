# v1.2.0-step-t — Certificates runtime refactor (2026-06-05)

First release in the `docs/release-notes/` convention. Step T
ships a live-data refactor of the `/certs` page: every cell
on the unified Domaines table reflects the real cert pool in
near real time (event-driven from certmagic), and the
managed-domain CRUD UX moves into a modal wizard that
matches the rest of the admin's add-flow patterns.

## Highlights

- **Unified Domaines table** — one row per cert with live
  runtime metadata (status, SAN count, issued / expires,
  issuer, ACME method). Replaces the pre-T.4 two-section
  layout (Managed domains + TLS-enabled routes), which only
  surfaced the configured shape and never reflected actual
  cert state.
- **Status badges with tooltip** — the locked
  `--status-up / --status-warn / --status-down / neutral`
  palette (visual consistency with Topology + Routes).
  `OBTAIN_FAILED` rows carry a hover tooltip with the last
  certmagic error message + timestamp.
- **Tabs filter** — `Tous` (default) / `Wildcard` /
  `Expirent bientôt` (within 30 days of expiry OR already
  expired). Pure client-side filter on the single fetched
  dataset.
- **Reframed "Politiques wildcard par apex" section** —
  rename from "Managed domains" to the semantically honest
  name. Declaration form hoisted into a modal wizard
  launched by a header `+ Wildcard apex` button.
- **OBTAIN_FAILED entry handling** — UI no longer surfaces
  cosmetically wrong values for never-obtained certs ("il y
  a 2 027 ans" / "expiré" rendered as `—`; KPI breakdown
  now sums to the total).
- **Tracker purge on managed-domain delete** — deleting an
  apex via the wizard immediately removes the corresponding
  rows from the Domaines table instead of leaving ghost
  entries until restart.

## Backend changes

- **New `internal/certinfo` package**: `Tracker` holds an
  in-memory `CertRuntimeInfo` cache populated from two
  sources — certmagic events ingested via a new Caddy
  events handler module (`arenet_cert_info`, registered
  with module ID `events.handlers.arenet_cert_info`), and
  boot-time on-disk reconcile from certmagic's
  `FileStorage` layout at `caddy.AppDataDir()`.
- **`GET /api/certificates`** — read endpoint returning the
  sorted-by-`notAfter`-ascending snapshot. AC #13 degraded
  mode: nil tracker returns `200 []` rather than `5xx`.
- **`Tracker.Remove(domain)`** — purge hook the
  `DELETE /api/settings/managed-domains/{apex}` handler
  calls after a successful caddy reload. Necessary because
  certmagic / Caddy v2.11.3 emit no cert-removal event
  (verified empirically against the vendored sources).
- **Forward-compat seam**: `Tracker.Subscribe(handler)` lets
  a future ACME-events-log consumer (Step T+1) attach
  without rewriting any T storage code. AC #18.
- **Boot-time visibility**: `cmd/arenet` logs
  `purger_present=<bool>` at boot so any future wire-up
  regression is immediately visible in journalctl instead
  of silently no-opping the purge path.

## Frontend changes

- **`/certs` page refactor** — KPIs are now live (computed
  from the cert list each render), table is unified, inline
  declaration form moved to the wizard. The stale "Per-cert
  runtime metadata is not exposed" banner is gone (the data
  now IS exposed). The "TLS-enabled routes" section is gone
  (merged into the unified Domaines table).
- **New `lib/utils/certificate-format.ts`** — pure helpers
  for the status → Badge variant mapping, source / challenge
  inference, expiry-day computation, dominant-issuer
  reduction, zero-time guarding. Full test coverage.
- **New `lib/components/certs/WildcardApexWizard.svelte`** —
  modal wizard wrapping the Pack A declaration form. Same
  wire contract; behavior changes only the UX shell.
- **Wire contract preserved** — the wizard POSTs to the
  same `settingsApi.createManagedDomain` endpoint with the
  same payload shape; the delete-with-revertTo modal is
  carried forward verbatim from Pack A.

## Known limitations

- **Manual force-renewal is not available**. T.2 (force-
  renew endpoint) and T.3 (force-renew storage + rate
  limit) were deferred — `caddytls.TLS.getConfigForName` is
  unexported in Caddy v2.11.3 and the workarounds (vendor
  patch, local fork, storage-delete) have unacceptable
  trade-offs for current bandwidth. certmagic's automatic
  exponential retry covers all real failure cases. Full
  recon + rationale in `docs/step-t-spec-amendment.md`;
  revisit conditions tracked under
  `#R-CERTS-force-renew` in `docs/backlog-step-t.md`.
- **ACME challenge sub-line** under the DOMAINE column is a
  UX heuristic, not a backend-supplied field
  (`#R-CERTS-challenge-heuristic` in the backlog). The
  heuristic is operator-correct in 100% of AreNET-
  configured paths today.
- **Periodic tracker reconcile** is not implemented
  (`#R-CERTS-reconcile-from-managed-domains`). The
  per-handler purge covers the only mutation path the
  current API exposes; if a future bulk-import or backup-
  restore endpoint lands, the periodic reconcile will be
  needed to handle the gap.

## Migration notes

- **No config change required** — the new layout ships
  automatically on update.
- **`/certs` URL is unchanged** — operators landing on
  bookmarks will see the new content without redirect.
- **No backend wire change** — the
  `/api/settings/managed-domains` endpoints retain their v1
  vocabulary and payload shapes. The "managed domain"
  terminology is preserved on the wire even though the
  frontend now calls them "policies".
