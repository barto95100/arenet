# v1.5.0-step-v1 — Normal traffic monitoring on the threat map (2026-06-07)

Step V.1 closes the 5th-color gap on the `/map` page.
Step V (v1.4.0) shipped a 5-category arc taxonomy
(`normal / throttle / waf / crowdsec / auth`) but only
the 4 threat sinks emitted — `normal` was reserved in
the legend with an "(à venir)" italic marker (commit
`2319699`). V.1 builds the backend pipeline for the
`normal` category: when an operator sets
`ARENET_NORMAL_TRAFFIC_SAMPLE_PCT > 0`, successful user
traffic (2xx + 3xx, excluding 304/HEAD/OPTIONS) emits a
green arc from the source IP location to the Arenet
position marker, alongside the existing threat colors.

The intent is **observability, not security**: V.1
doesn't change request-path behavior, doesn't block
anything, doesn't add headers. Operators get a baseline
of "this is what your traffic looks like when nothing
is wrong" — useful for comparing against incident
patterns.

## Highlights

- **New `geo.NormalSink` interface + `DefaultNormalSink`
  implementation** in `internal/geo/normal_sink.go`. 3
  gates apply in sequence before MMDB lookup runs:
  random sample (math/rand/v2 PCG, synchronized),
  RFC1918 short-circuit (LAN sources skip emission per
  spec §3.8), per-IP cooldown LRU (`container/list`-
  backed, ~70 LOC hand-rolled, 4096-entry default
  capacity). Sample-before-lookup saves the 1-5 µs warm
  / 50-200 µs cold MMDB cost on the dropped 95%.
- **Wire-up at the existing `arenet_routemetrics`
  middleware** (`internal/metrics/middleware.go`). 4
  upstream eligibility gates short-circuit before the
  sink: D1 status class + method, D3 hardcoded path
  exclusion + operator-configurable extension. The
  hardcoded list includes `/api/v1/ws/geo-events` —
  pre-empts the meta-recursion where opening `/map`
  upgrades a WS that would itself emit a normal event
  the operator sees as their own connection arc.
- **5th cross-cutting forwarder** (`cmd/arenet/geo_forwarders.go`):
  `geoForwardingNormalSink` wraps the production sink
  behind the `metrics.NormalSubmitter` interface. The
  wrapper is intentionally a passthrough — the sink
  owns sampling/cooldown/LAN/publish — kept for API
  symmetry with the 4 existing forwarders + future-
  proofing for a V.1.X audit/log/metric hook on the
  success path.
- **`atomic.Pointer[T]` globals** for `NormalSubmitter`
  + `ClientIPFn`. V.1.2 originally cached at Provision
  time, but the V.1.1 sink + V.3 geoBus + V.2
  geoEnricher are all constructed AFTER `mgr.Start` in
  `main.go`, so Provision sees nil and any late install
  would silently no-op. V.1.3 migrated to lock-free
  per-request reads — nanoseconds per Submit-eligible
  request, no measurable cost on the V.1-disabled path
  (nil short-circuits before the eligibility gates).
- **3 new env vars** (per spec §5). All default to safe
  values; SAMPLE_PCT=0 is the canonical disabled state
  (no events, no LRU touch, no MMDB lookups). Invalid
  SAMPLE_PCT / cooldown values are FATAL at boot —
  silent fallback would mislead an operator who set the
  var thinking they enabled the feature.
- **`caddymgr` threading** of the operator-supplied
  exclude paths into every per-route metricsHandler
  JSON emit. `SetNormalTrafficExcludePaths` mirrors
  the `SetCrowdSecConfig` pattern (called before
  `mgr.Start`); the per-route field is `omitempty` so
  byte-equality with pre-V.1.3 configs is preserved on
  default installs.
- **Boot signal** ("normal traffic sink wired") emits
  the active config dimensions (`present`,
  `sample_pct`, `cooldown`, `exclude_paths_count`,
  `hardcoded_exclude_paths_count`). Visible in
  journalctl whether V.1 is enabled or disabled so
  operators can confirm the state.
- **Legend "(à venir)" marker dropped** from the
  `normal` row in `web/frontend/src/lib/components/Map/
  MapLegend.svelte`. Arc rendering, color application,
  WS reception, and replay all work uniformly across
  the 5 categories already (proven by V.6's per-
  category-color parametrized test). The `comingSoon`
  flag + `.legend-coming-soon` CSS class + the
  `{#if entry.comingSoon}` conditional stay in place
  for future category use.

## New operator config surface

| Env var | Type | Default | Notes |
|---|---|---|---|
| `ARENET_NORMAL_TRAFFIC_SAMPLE_PCT` | int `0..100` | `0` | **Master switch**. `0` = V.1 disabled (no events, no LRU touch, no MMDB lookups). `1-100` = random sample percentage. Invalid values FATAL at boot. |
| `ARENET_NORMAL_TRAFFIC_PER_IP_COOLDOWN` | Go duration | `30s` | Per-IP rate cap. `0s` (or any non-positive duration) disables the cooldown gate (random sample alone bounds the rate). |
| `ARENET_NORMAL_TRAFFIC_EXCLUDE_PATHS` | CSV string | `""` | Extends (does NOT replace) the hardcoded `/healthz`, `/metrics`, `/api/v1/ws/topology`, `/api/v1/ws/geo-events` list. Whitespace around commas trimmed; empty entries skipped; duplicates deduped. Prefix-match semantics. |

Interaction semantics:

- `SAMPLE_PCT=0` short-circuits everything — the other
  two vars are read but have no effect.
- `SAMPLE_PCT=100 PER_IP_COOLDOWN=0` emits EVERY
  eligible request (load-test scenario; documented but
  not recommended at >100 req/s sustained).
- `SAMPLE_PCT=100 PER_IP_COOLDOWN=30s` emits at most
  one event per unique IP per 30 s. Useful for "show me
  which countries are reaching me" without volume.

The smoke configuration on AreNET-test used
`SAMPLE_PCT=10`; green arcs rendered correctly from
real remote traffic, no observable backend overhead,
no frontend frame drops.

## Architecture changes

- **`internal/geo/normal_sink.go` (NEW, ~355 LOC)**:
  `NormalSink` interface + `DefaultNormalSink` impl
  with the 3-gate sampling/RFC1918/cooldown pipeline,
  hand-rolled `ipCooldownCache` (LRU+TTL over
  `container/list`), `NoopLANCounter` for backend-only
  tests + the V.1.4 LAN-counter deferment.
  `EnrichNormal` added to `internal/geo/enricher.go`
  for the wire-shape symmetry.
- **`internal/metrics/middleware.go`**:
  `NormalSubmitter` + `ClientIPFunc` interfaces locally
  defined (no `internal/geo` import — keeps the metrics
  package dep graph narrow; production type satisfies
  the interface structurally). `eligibleForNormal`
  gate function. `HardcodedExcludePaths()` public
  accessor (returns a defensive copy) for the V.1.3
  boot signal. `RouteMetricsHandler.normalSink` +
  `clientIP` fields DROPPED — live reads from atomic
  globals per request instead.
- **`internal/metrics/global.go`**: globals migrated
  from `sync.RWMutex` (V.1.2) / `sync.Once` to
  `atomic.Pointer[NormalSubmitter]` +
  `atomic.Pointer[ClientIPFunc]`. Lock-free
  per-request reads; both setters are re-callable so
  runtime config swaps (deferred) become possible
  without re-architecting.
- **`cmd/arenet/normal_traffic_config.go` (NEW, 125
  LOC)**: 3 env-var parsers with strict validation
  (SAMPLE_PCT clamp + integer reject, PER_IP_COOLDOWN
  Go duration, EXCLUDE_PATHS forgiving CSV with
  trim/dedup).
- **`cmd/arenet/geo_forwarders.go`**: 5th wrapper type
  `geoForwardingNormalSink` (passthrough).
- **`cmd/arenet/main.go`**: env parsing +
  `mgr.SetNormalTrafficExcludePaths` BEFORE
  `mgr.Start` (so the first applyLocked emits the
  operator's exclude paths on every route's
  metricsHandler). `geo.NewDefaultNormalSink` +
  `metrics.SetNormalSubmitter` AFTER `geoBus +
  geoEnricher` exist. `metrics.SetClientIPFn` AFTER
  `ipExtractor` exists. Boot signal HF4-pattern.
- **`internal/caddymgr/manager.go`**:
  `SetNormalTrafficExcludePaths` setter mirroring
  `SetCrowdSecConfig`, threaded through `buildOpts`
  into the per-route `metricsHandler["exclude_paths"]`
  JSON emit. `omitempty` preserves byte-equality with
  pre-V.1.3 configs on default installs.
- **`web/frontend/src/lib/components/Map/MapLegend.svelte`**:
  single-line `comingSoon: true` removal on the
  `normal` row. Conditional surface preserved for
  future category use.

## Known limitations / deferred

- **LAN counter deferment** (`#R-NORMAL-TRAFFIC-lan-counter`):
  spec §3.5 D2 specified that RFC1918 sources count in
  the V.6 LAN pill. V.1 ships `NoopLANCounter`. LAN
  normal traffic is health checks + monitoring polls +
  LAN agent telemetry — not actionable to operators.
  The V.6 LAN pill was designed for security signal
  visibility (auth-failure spikes = lateral movement;
  WAF blocks = misconfigured internal service); normal
  LAN traffic would dilute that signal. Wiring backend
  → frontend LAN counter for normal traffic also
  requires a new HTTP endpoint or a WS frame type
  extension that didn't fit V.1.4 scope. Explicit
  deviation from spec §3.5 D2 — V.1.X revisits if
  operator feedback shows demand.
- **Per-route opt-in deferred** (`#R-NORMAL-TRAFFIC-per-route`):
  D6 keeps V.1 at a global toggle. Per-route opt-in
  would add a route-schema column (DB migration) +
  admin UI checkbox + `caddymgr` config-emit branch.
  V.1's sampling already bounds total volume; per-
  route filtering is a precision knob that doesn't
  gate v1 ship.
- **Static asset URL coalescing NOT implemented**
  (`#R-NORMAL-TRAFFIC-asset-coalesce`): D4 relies on
  sampling + per-IP cooldown to bound noise. If a page
  load produces 10-20 events across different routes
  that the cooldown doesn't suppress, the operator
  may see a visible burst. Future work: per-(IP, route)
  cooldown OR a MIME-type filter on Content-Type.
- **Sampling is per-process**
  (`#R-NORMAL-TRAFFIC-multi-instance`): in a
  multi-instance Arenet deployment (Step X scope),
  each instance samples independently. Two instances
  at `SAMPLE_PCT=5` behind a load balancer produce
  ~10 % combined emission rate (assuming round-robin),
  and per-IP cooldowns are per-instance — the same IP
  hitting both instances within 30 s emits twice.
  Acceptable for v1 (single-instance is the homelab
  target); coordination via a shared Redis cache
  deferred.
- **No "hide normal arcs" frontend toggle**
  (`#R-NORMAL-TRAFFIC-hide-toggle`): D8 ships green
  arcs alongside the existing 4 categories. During a
  security incident an operator may want to focus on
  threats only. Future polish: a per-category
  visibility toggle in the legend.
- **3xx redirects emit as normal** — Following the V.1
  spec §D1, 301/302/307/308 all pass the gate. An
  operator running a redirect-heavy front (e.g.
  HTTP→HTTPS upgrade, www→apex canonical) will see
  green arcs that aren't "user-visible success" in
  the strict sense. Cosmetic; if the noise hurts,
  a future increment can move 3xx to a separate
  category or add a status-class filter knob.

## Migration notes

- **No config change required** — V.1 ships disabled
  by default. Upgrading from v1.4.0-step-v + HF3
  produces no behavioral change until the operator
  sets `ARENET_NORMAL_TRAFFIC_SAMPLE_PCT > 0`.
- **No DB migration** — V.1 is bus-only per spec §D7.
  No new `metrics.db` schema bump (current at v6
  post-V.2).
- **No frontend bundle size change** — V.1.4 is a
  single-line legend edit; the V.6 arc rendering
  pipeline was already category-uniform.
- **`/api/v1/ws/geo-events` exclusion is automatic** —
  operators don't need to add it to their
  `ARENET_NORMAL_TRAFFIC_EXCLUDE_PATHS` because it's
  in the V.1.2 hardcoded list. Same for `/healthz`,
  `/metrics`, `/api/v1/ws/topology`.
