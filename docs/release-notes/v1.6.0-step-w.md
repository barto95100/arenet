# v1.6.0-step-w â€” Country-block per-route gate (2026-06-08)

Step W ships the **first operator-declared enforcement layer**
in Arenet. Previous threat-mitigation layers (Coraza WAF
under Step I/M, hslatman/caddy-crowdsec-bouncer under Step
N) are signature-driven or feed-driven: an operator
enables them once, then trusts the upstream rule set or
LAPI decision stream to do the right thing. Step W adds a
**static policy lever the operator owns end-to-end**: a
per-route allow/deny list of ISO 3166-1 alpha-2 country
codes, evaluated at the Caddy edge before the proxy
chain reaches the upstream.

The use case is "I run Home Assistant on `ha.example.com`
and I only ever connect from FR" â†’ operator sets
`mode=allow` + `countryList=["FR"]` and every other source
country gets a 403/451/444 (operator-selectable) before
the request touches the auth gate, the WAF, the CrowdSec
bouncer, or the upstream. Symmetric "deny RU + KP + CN
without touching the FR/DE/IT traffic" via `mode=deny`.

Step W reuses the V.1 GeoIP City MMDB (no second database
to install, no ASN dimension â€” see "Known limitations"
below). The Caddy module is hand-rolled at
`internal/countryblock/` rather than vendoring
`porech/caddy-maxmind-geolocation` â€” the integration
depth (reuse of `geo.Lookup`, custom event emission,
trusted-IP allowlist, RFC1918 short-circuit) recoups the
~150 LOC of standard-module boilerplate.

## Highlights

- **New `internal/countryblock/` package** (W.1, commit
  `5ac5b78`). Caddy HTTP middleware
  `http.handlers.arenet_country_block` with a pure-Go
  `Evaluate` function exposing the 5 bypass paths
  (trusted-IP allowlist, RFC1918 / loopback / link-local,
  off-mode short-circuit, degraded-lookup fail-open per
  AC #18, then allow/deny match against the operator's
  country list). The matcher's stable reason enum
  (`"allow-match"`, `"deny-miss"`, etc.) ships verbatim
  through the event row so the W.5 activity log can
  render a tooltip without parsing free text.
- **Per-route storage + API** (W.2, commit `57394b5`).
  `Route.CountryBlock` field (mirrors `WAFMode` + `AuthMode`
  patterns) with the spec Â§D2 footgun rule enforced at
  the API layer: `mode=allow && countryList=[]` returns
  400 with `ErrAllowListEmpty` because the operator
  would otherwise block ALL non-RFC1918 traffic from
  their own route. Pre-W rows decode with zero-value
  `CountryBlock{Mode:""}` â€” Go's `encoding/json`
  auto-migrates missing keys, no BoltDB migration
  needed. PUT mirrors `wafMode`/`healthCheck`
  preserve-or-replace semantics (nil pointer = preserve,
  non-nil = full replacement).
- **`caddymgr` chain-position #2 emit** (W.3, commit
  `60b05d8`). The country-block handler slots BETWEEN
  the V.1.2 `arenet_routemetrics` middleware (#1, MUST
  observe the final status code) and the CrowdSec
  reputation gate (#3). Operator-declared static policy
  is **cheaper** than the dynamic CrowdSec LAPI lookup
  (~1 Âµs MMDB lookup + 2-byte country string compare
  vs ~10 Âµs LRU scan against a 100k-entry decision
  set) AND **more permanent** (op declares "block RU
  forever"; CrowdSec decisions expire with their TTL),
  so it fires first. The emit is omitted entirely for
  `mode=off` routes â€” zero per-request cost for routes
  that don't use the feature.
- **Operator-facing env vars + atomic.Pointer globals**
  (W.3). `ARENET_COUNTRY_BLOCK_STATUS` (default 403,
  accepts 403/451/444 â€” invalid values WARN at boot and
  fall back per spec Â§D3, never FATAL) and
  `ARENET_COUNTRY_BLOCK_TRUSTED_IPS` (CSV of CIDRs +
  bare IPs auto-expanded to /32/128 â€” malformed entries
  WARN per entry and drop, valid entries still apply
  per spec Â§D4). Sample percentage + cooldown REUSE the
  V.1.3 `ARENET_NORMAL_TRAFFIC_*` env vars to avoid
  env-var sprawl. Late-install resilience pattern (V.1.3
  precedent): a nil-inner Lookup placeholder is installed
  BEFORE `mgr.Start`, then atomically promoted to the
  real `geo.Lookup` once the MMDB opens later in boot.
- **Sink + 6th GeoEvent category + schema v7â†’v8** (W.4,
  commit `03e35f7`). `DefaultCountryBlockSink` mirrors
  the V.1.1 normal-sink shape (sample + cooldown LRU)
  but with **dual fan-out**: every block decision
  publishes to the geo bus (live `/map` gray arcs) AND
  enqueues a `country_block_event` row for the 30-day
  retrievable activity log. The W.1 sink seam widened
  from a 4-arg `Submit` to `Submit(BlockMatch)` so the
  persisted row carries `Mode + Reason` in dedicated
  columns (not a free-text Details blob â€” the W.5
  frontend filters on them directly).
- **`country_block_event` schema v7â†’v8**. New table with
  `(ts, route_id, src_ip, country, mode, status_code, reason)`
  + 2 indexes (`(ts)`, `(route_id, ts)`). Retention pruner

  WAF/throttle/auth/decision horizons â€” country-block is

- **6th GeoEvent category** (W.4). `CategoryCountryBlock =

- **Frontend UI** (W.5, commit `7cd0dbd`). Three surfaces:

- **New observability API endpoint** (W.5).

## Env vars

| Var | Type | Default | Behavior on invalid |
|---|---|---|---|
| `ARENET_COUNTRY_BLOCK_STATUS` | int | `403` | WARN + fall back to 403 (boot continues) |
| `ARENET_COUNTRY_BLOCK_TRUSTED_IPS` | CSV CIDR/IP | empty | per-entry WARN + drop, valid entries still apply |
| `ARENET_NORMAL_TRAFFIC_SAMPLE_PCT` | int 0..100 | `0` | reused â€” controls country-block emission too |
| `ARENET_NORMAL_TRAFFIC_PER_IP_COOLDOWN` | duration | `30s` | reused â€” same per-IP cooldown LRU |

The country-block sink runs at `SamplePct=0` (the
default â€” no events) until the operator opts into normal-
traffic monitoring. When they do (typically `SAMPLE_PCT=5`
for moderate visibility), country-block events flow
through the same sampling/cooldown pipeline as normal
traffic. Splitting into category-specific env vars is
deferred (`#R-W-env-sprawl-vs-tuning`).

## Schema migration

- **v7 â†’ v8** (`migrateV7toV8` in
  `internal/observability/migrate.go`). Adds the

  migration idempotent under partial-failure replay.

## Known limitations / deferred

- **No country-name autocomplete in the chip input**
  (`#R-W5-country-name-autocomplete`). Operator types
  `FR`, sees a chip rendered as `FR`. A full alpha-2 â†’
  name dictionary (250 codes) would add ~5 KB to the
  bundle for marginal UX gain â€” operators reading the
  chip already know what those codes mean. Defer to a
  future round if operator feedback warrants it.
- **No RouteID â†’ host resolution in the activity log**
  (`#R-W5-resolve-host-in-logs`, **priority 1 per W.6
  smoke feedback**). The W.4 schema intentionally omits
  `host` from the persisted row (the W.3 caddymgr emit
  elided it from the Caddy config). The `/logs` row
  currently shows the raw `route_id` UUID in the path
  column â€” Gate 9 of the W.6 smoke matrix surfaced
  this as a real operator-confusion moment ("which
  route is `2db59ddf-...`?"). A future increment can
  populate a routeId â†’ host map at page mount (via the
  existing `fetchRoutes` call) and resolve per row.
  Documented in source on `mapCountryBlock`. Promote to
  priority 1 in the post-W backlog.
- **No `caddy.Validate`-driven empirical test for the
  country-block handler** (`#R-COUNTRYBLOCK-add-loadscleanly-in-manager_test`).
  W.3 attempted one but it polluted
  `TestSyncRegistry_NotCalledOnReloadFailure` (file
  ordering: my new test ran BEFORE manager_test.go,
  leaving Caddy admin endpoint + tls cache maintenance
  goroutines running). Compromise: shipped a substring-
  assertion JSON-shape test instead. The full
  `caddy.Validate` guard belongs inside `manager_test.go`
  (where ordering puts it after the state-sensitive
  test) in a future commit.
- **No GeoLite2-ASN dimension** (`#R-COUNTRYBLOCK-asn-deferred`).
  Spec brief mentioned `ASN` on the `BlockEvent` shape;
  the implementation skipped it. V.1's MMDB is the City
  DB; ASN would require the separate
  `GeoLite2-ASN.mmdb` install path + a second
  `geo.Lookup` instance + env var. The W.5 activity log
  surface for ASN ("blocked by autonomous system X")
  has unclear operator value at homelab scale.
  Documented as deferred in
  `internal/countryblock/module.go` BlockMatch
  doc-comment.
- **No dry-run mode** (`#R-COUNTRY-BLOCK-dry-run`).
  Operators can't `mode=dry-run` to see WHAT WOULD be
  blocked without actually blocking. Workaround:
  enable `mode=deny` with a single test country
  reachable from a known VPN, watch the activity log
  for 5 minutes, decide whether to commit.
- **Country-by-IP is imperfect**
  (`#R-COUNTRY-BLOCK-vpn-mobile-cgnat`). VPN exits,
  mobile carriers, and CGNAT pools all routinely
  misattribute country. Operators MUST NOT rely on
  country-block for legal compliance â€” only as a noise
  filter / first-line gate. Will document this in the
  operator-facing doc at W.7 if/when one ships.
- **WAF + WorldMap fixes shipped intermediate**
  (`v1.5.1-waf-bugfix` + `v1.5.2-waf-websocket-bypass`).

## Migration notes

- **No config change required**. Step W ships at
  `mode=off` for every existing route (zero-value Go
  decode of the new `Route.CountryBlock` field).
  Upgrading from `v1.5.2-waf-websocket-bypass` produces
  no behavioral change until the operator picks a
  per-route mode via the UI.
- **No BoltDB migration**. The `Route.CountryBlock`
  field decodes as zero-value `Config{Mode:""}` on
  pre-W rows (per Step J.2's healthcheck precedent â€”
  Go's `encoding/json` zero-fills missing keys). The
  W.2 storage validator accepts `Mode=""` as a synonym
  for `"off"`.
- **`metrics.db` v7 â†’ v8** is automatic on first boot.
  The `country_block_event` table is created idempotently
  via `CREATE TABLE IF NOT EXISTS`. Existing rows in
  other tables are untouched.
- **Reuse of V.1 sampling env vars** means an operator
  who set `ARENET_NORMAL_TRAFFIC_SAMPLE_PCT=5` for
  normal-traffic monitoring AUTOMATICALLY gets 5 %
  sampling of country-block events too. To opt out of
  country-block emission specifically while keeping
  normal-traffic monitoring, set `mode=off` on every
  route â€” the W.3 caddymgr emit then skips the handler
  and no events fire.
